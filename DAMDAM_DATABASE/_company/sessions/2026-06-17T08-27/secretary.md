### 📋 파트 1: [Designer] PSD 원본 에셋 제출 필수 점검 항목

| 체크리스트 | 내용 | 준수 여부 | 담당자 | 비고 및 기준 |
| :--- | :--- | :--- | :--- | :--- |
| **① 규격/레이어** | 1080px x 1350px 고정 원본 크기 유지 (필수) | [ ] 완료 | Designer | 모든 에셋에 적용. |
| **② 레이어 분리** | 모든 요소(텍스트, 이미지, 공식 컴포넌트 등)는 개별 레이어로 완벽히 분리되어야 함. | [ ] 완료 | Designer | 그룹화된 텍스트 박스는 불가. |
| **③ 파일명 규격** | `P[페이지]_[요소유형]_[순서].psd` (예: P03_formula_01.psd) 준수. | [ ] 완료 | Designer | 최종 납품 시 반드시 이 규칙을 지킬 것. |
| **④ 핵심 컴포넌트** | 공식화된 '핵심 개념 컴포넌트' 라이브러리(공식, 체크리스트 템플릿 등)를 최신 버전으로 사용했는지 확인. | [ ] 완료 | Designer | #FF6B6B 강조색 적용 여부 필수 검토. |
| **⑤ CTA 유연성** | CTA 영역은 반드시 최종 Mockup 단계에서 수정 가능한 '여백/강조색 분리 레이어'를 확보해야 함. | [ ] 완료 | Designer | 텍스트가 고정되지 않도록 원본 구조화 필요. |

### 🎧 파트 2: [Editor] 오디오 트랙 및 Mockup 통합 점검 항목

| 체크리스트 | 내용 | 준수 여부 | 담당자 | 비고 및 기준 |
| :--- | :--- | :--- | :--- | :--- |
| **① 싱크(Sync)** | 웹툰 컷 전환, 중요한 논리적 포인트 강조 시 오디오 트랙의 타이밍이 정확하게 일치하는가? | [ ] 완료 | Editor | 사운드 이펙트 위치 체크. |
| **② 메시지 전달력** | 스크립트의 핵심 키워드가 목소리 톤과 속도에 명확히 반영되었는가? (너무 빠르거나 느리지 않은가?) | [ ] 완료 | Editor | 공감보다 '논리적 해석'이 강조되어야 함. |
| **③ 플랫폼 적합성** | 오디오 트랙만으로도 웹툰의 흐름과 감동(혹은 논리)이 충분히 전달되는지 검토. (자막 의존도를 낮출 것). | [ ] 완료 | Editor | 릴스/숏폼 환경에 최적화된 길이와 밀도가 필요함. |

### ✨ 파트 3: 최종 통합 승인 체크포인트 (Gate Review)

1.  **[Designer]**: 모든 자산이 **V4.0 규칙**을 따르는지 검증 완료.
2.  **[Editor]**: 오디오 트랙과 Mockup 프리뷰를 결합하여 테스트 실행 및 오류 지점 발견.
3.  **[Secretary/PM]**: 최종 콘텐츠 캘린더와 마케팅 문구를 조합하여 배포 로드맵 확정 (최종 승인).
