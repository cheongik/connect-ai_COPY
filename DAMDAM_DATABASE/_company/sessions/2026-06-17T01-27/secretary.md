### 📂 1. 최종 자산 정리 폴더 구조 (Final Asset Folder Structure)

모든 프로젝트의 자료는 하나의 루트 폴더 아래에 체계적으로 분류하는 것이 중요합니다. 현재 진행 중인 '공감 과부하' 콘텐츠를 기준으로 폴더를 구성했습니다.

**[Root Path]**
`~/DAMDAM/Assets_Library/[YYYYMMDD]_Webtoon_CBH` (예: `~/DAMDAM/Assets_Library/20260617_Webtoon_CBH`)

**[Sub-Directory Structure]**
```
├── 📁 01_Script & Strategy (Writer 담당)
│   ├── 📄 Storyboard_Final.pdf      // 최종 스토리보드 확정본
│   ├── 📄 Copywriting_Full.docx     // 전체 페이지별 스크립트 원문
│   └── 📄 CTA_Guide_V2.0.pdf        // 최종 Call-to-Action 가이드라인 (폰트, 문구 등)
|
├── 📁 02_Design & Mockup (Designer 담당)
│   ├── 📂 Webtoon_Pages_Source (PSD 원본 파일 저장소)
│   │   ├── Webtoon_CBH_P1_V3.psd    // 페이지 1 PSD
│   │   └── ...
│   ├── 📄 Design_Guideline_V2.0.pdf  // 최종 아티스트/디자인 규격 가이드라인
```

### 🏷️ 2. 필수 파일명 규격 (Naming Convention)

파일 이름을 체계화하여 검색 용이성과 버전 관리를 최적화합니다.

**[규칙]** `[콘텐츠 종류]_[주제 약자]_[단계/버전]_[YYYYMMDD].확장자`

*   **예시 1 (웹툰 페이지):** `Webtoon_CBH_P3_V2_20260617.psd`
    *(의미: 웹툰 / 공감과부하(CBH) / 페이지 3 / 버전 2 / 2026년 6월 17일)*
*   **예시 2 (최종 스크립트):** `Script_CBH_Final_20260617.docx`

### ✅ 3. 납품 전 점검 체크리스트 v2.0 (Pre-Delivery Checklist)

담당 에이전트가 최종 산출물을 제출하기 직전에 이 리스트를 반드시 확인하고, 누락된 항목이나 규격 오류가 없는지 검토해야 합니다.

| No. | 필수 자산/체크 사항 | 담당자 | 요구 사양 및 형식 (Specification) | 마감 기한 (Deadline) |
| :---: | :--- | :---: | :--- | :---: |
| **A** | **[스크립트] 최종 스토리보드 확정본** | Writer | PDF 또는 PPT. 컷 단위의 명확한 분할 및 전체 흐름(Flow) 포함 필수. | T-3 Days |
| **B** | **[스크립트] 페이지별 카피라이팅 원문** | Writer | DOCX (최종 승인된 버전). 이미지에 삽입될 모든 텍스트를 별도 목록화. | T-2 Days |
| **C** | **[디자인] 최종 아티스트 가이드라인** | Designer | PDF. PSD 레이어 구조, 캐릭터/배경 에셋 활용법, CTA 영역 배치 규격 명시. | T-3 Days |
| **D** | **[디자인] 웹툰 원본 페이지 (PSD)** | Designer | PSD 파일. 모든 컷은 레이어가 분리되어야 하며, 고해상도(300dpi) 필수. | T-1 Day |
| **E** | **[오디오] 배경음악/효과음 트랙** | Editor | MP3 또는 WAV (원본). 페이지의 감정 변화에 맞춘 믹싱 버전 제공. | T-1 Day |
| **F** | **[오디오] 내레이션 녹음본 (Clean)** | Editor | WAV (고품질 오디오). 기계적 잡음 없이, 배우가 읽은 스크립트와 완벽 동기화되어야 함. | T-1 Day |
| **G** | **[통합] 최종 납품 승인 완료 보고서** | Yeongsook | 이 체크리스트에 대한 "승인(Approved)" 사인 및 전체 로드맵 공유. | D-Day Morning |

***
*T = Target Date (콘텐츠 배포 예정일)*
*D-Day = 실제 콘텐츠 업로드/배포 당일*
